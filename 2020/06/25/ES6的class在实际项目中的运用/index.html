本文字数： 6.1k 阅读时长 ≈ 6 分钟
摘要：ES6的class在实际项目中的运用，包括私有属性，静态属性，公有方法，以及一个很好的系统架构微总结

前言

ES6的class 的基本语法，我感觉自己说不好，可参阅阮一峰老师的文章，有详细讲解。

目前主流的MVVM架构的框架主要有3 个：VUE, REACT, ANGULAR。而现在工作市场上应用最多的是VUE 和REACT。过去几年，我一直靠VUE 行走江湖，对另外两个敬而远之。但目前工作的这家公司全产品线用REACT，加上有一个好的前端Leader，我也开始拥抱REACT 。

最近，我们做的一个项目是CRM 系统，前端Leader 凭借超强的实力，以函数式编程，ES6，RESTFULL（把此风格结合path-to-regexp插件运用到路由相关的业务逻辑管理中来，是最让我震撼的），UMI，UMI HOOK, REACT HOOK，设计模式 等技术娴熟联合搭建起一个高健壮，高扩展，高解耦，无冗余，可插拔式系统架构，整个项目从一二三级路由，FORM，TABLE，权限，状态管理，业务组件，公共组件等全方位实现配置式编程，大大提高了迭代开发效率，让我佩服的五体投地。此项目开发期间，我涨了很多姿势，哈哈，回过头再去翻看那些代码，就像看到一个集天使般的脸蛋，魔鬼般的身材，学富五车的才华于一身的小姐姐，赏心悦目的感觉。想起以往新入职一家公司，给别人擦屁股的痛苦，心中顿时一句MMP 不知当讲不当讲。

这里在不涉及公司商业机密的前提下，总结学习一个很小的部分，就是FORM 和 TABLE 的控制生成器。
