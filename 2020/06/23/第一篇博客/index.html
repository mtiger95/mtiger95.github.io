创建技术博客

发表于 2020-06-23 更新于 2020-06-26 分类于 情感
本文字数： 1.7k 阅读时长 ≈ 2 分钟
摘要：建立博客

前言

今天花了半天时间，终于把这个技术博客跑起来了，也遇到了一些小坑，不过多看看文档，查些资料都还是解决了。这篇文章写的很不错，官方文档讲的比较粗犷，我通过这篇文章设置了主题，背景，外部链接，小宠物等等，在此感谢作者。前几年，一直在博客园上写和复制别人的文章，不过感觉就像时代的变迁，像从jquery 到Vue，像从多页面到SPA ，于是选择了这个灵活强大的几乎纯配置型的博客。想过从零手写搭建，这样当然更能把这几年的所学所用，更加系统的串联起来。然而，我更清楚，自己创建博客的初衷，是为了记录和分享日常工作中所想所悟所得的干货，那些更基础，更底层，更刨根问底的原理知识，一般都是看阮一峰、张鑫旭等等老师的文章，细腻易懂且不会像官方文档那般枯燥，于是，‘拿来主义’的想法便占了上风。博客运行起来后，我更是窃喜，自己选择是对的，不然，自己利用业余时间设计，搭建，怎么也得一个月不止了，嘿嘿，还是把时间花在刀刃上吧。

立个FLAG

立Flag 是一件容易的事，但能坚持执行却不是每个人都能做到了。也许是因为工作很忙，加班到很晚；也许是周末了想陪陪家人或朋友，出去游玩散心。
总之，我们可以为自己不想做某件事找千百个借口，哈哈哈哈。 比如，马上就是传统节日端午节了，少不了亲戚朋友聚聚，聊聊工作生活，外出走走。
也是刚刚好，行政美女把公司发的李子柒网红粽子递到我手上后，我最后一个小问题也解决了，博客正常运行起来，也记住了几个常用的命令。以后会尽量多写写文章，安静的思考，冷静的用文字表达，在这块属于我的田野上，让自己思绪飞扬起来，不用像日常的社交那样有诸多顾虑和压力了。

结尾

前端技术因为交互终端的多样化，应用场景的多变，同时基于互联网的快速发展，催生了众多技术栈，多而杂，随便举几个例子

  • PC
  • PAD
  • android mobile
  • ios mobile
  • wechat app(当然还有支付宝小程序、字节系、京东系)
  • PWA
  • SSR
  • (H5 | Hybrid | Native) App
  • 数据可视化

这仅仅只是交互终端的多样，还不包括繁多的技术栈。可能我上面这句话说的不是很准确，一切还是以市场为主导的，技术是一直在突破和发展着。。。

前端最重要的根基是javascript，其它的主流MVVM框架，像VUE，REACT，除了语法和思想，在实际工作中，写的更多的还是原生JS脚本，ES6一定要熟悉，新增的数据结构和API要熟练应用，前些天帮我们组前端Leader 一面了一个6年多的前端小哥，一开始他自信到自负，以至于在回答我的问题时，拿着笔在空中乱舞，当我问了JS基础的问题，像Map 和 Object 的区别，数据类型隐式转换，事件模型，EventLoop，tree Data 转换，和class 类等几个基础问题时，他哑口了，于是他在气势上也败下阵来，把笔轻放在桌子上，认真起来。但看他项目经验丰富，框架玩的溜，我还是让他过了一面，但在Leader 那里还是没能通过，好的公司和Leader 并不会问太多框架的东西，因为JS基础才是编程和解决问题的根基，当然还有其它的许多点，像设计模式，工程化，但追踪到底，还是离不开基础。

始终觉得前端的技术体系多而杂，而且轮子多，框架多，VUE3，FLUTTER, TYPESCRIPT 这些新主流也接踵而来，当我刚开始学习Node 时，Deno 又开始摇旗呐喊；Serverless架构也会是必然，从物理机到云主机，到Serverless架构，去服务器化是越来越明显，上云也是必然，只是什么时候上的问题，早上早踩坑，同样也是早享受。新技术栈仿佛在建立一座座收费站，不断收割着我们前端er 的时间和精力。学的越多，心里越虚，感觉自己貌似什么都会一点，又什么都不会。

不过，抱怨是没有任何作用的，徒增压力而已，以市场为主导，以应用场景和产品为方向，正确选择合适的技术栈，针对性的学习。才不会让我们在琳琅满目的前端技术栈中迷失。

以后，可能不会仅仅只写技术文章，也会写写生活中的经历和感想吧。。

后天是2020年6月25日，农历五月初五，端午节，纪念屈原的日子。放假3天，祝所有读者和自己，工作顺利，生活顺心！
