React中自动生成Form表单控件的组件的封装

发表于 2020-06-29 分类于 React
本文字数： 7.5k 阅读时长 ≈ 7 分钟
摘要：React中通过函数组件封装一个能自动生成Form表单控件的组件，只需传入相关配置即可，可在任何以Form表单为搜索条件，以Table为展示数据的页面使用。

目的

在接手的许多项目中，不论Vue 还是React 的，特别是Vue 的项目，Form 表单的元素基本上都是一个一个从UI 框架上搬过来，然后进行双向绑定。那么页面多了，用的Form 表单就多了，就要写很多次各种表单元素，针对不同页面不同Form ，其实有一种更好的方式来实现。

我在上一篇文章ES6的class在实际项目中的运用中有提到一个Form表单封装组件，在这篇文章作一次详细讲解，一般我们开发的B 端管理系统，像CRM, ERP, SCM 这些，少不了会有许多表格来展示各种维度的数据，页面构成也基本一样，就是上面一个Form 表单做搜索条件，下面一个Table 做数据展示。那我们就可以进行组件封装，以免不停做重复工作。

基于React 官网介绍和推荐，文章中都采用函数式组件结合React HOOK。

思路

  • 纵观90% 以上的这种页面的Form 表单控件，其表单元素都是有这几个构成因子：

    • label: 表单元素前面的展示文字，表述这个元素代表的条件
    • type: 表单元素类型，比如input, select, datepicker, 还有像城市这种Cascader级联选择框，或者是关系型联动下拉选择框
    • key: 表单元素的键，就是我们要传给后台的键
    • options: 如果是一个select 元素，要传给元素的下拉选项
    • inputFormat / outputFormat: 这个可以是自己添加的因子，代表form 表单控件传入传出的数据的format 处理方法

  • 由上可以看出要传给Form 控件封装组件的第一个配置了（在后面会用到）：

  • Form 表单控件的表单元素相关构成知道了，那其它的就都是一些常规配置项了，比如初始默认值、有的表单元素可出现清空按钮、对国际化语言包的支持, 控件的Submit 事件的派发这些。下面就开撸！
