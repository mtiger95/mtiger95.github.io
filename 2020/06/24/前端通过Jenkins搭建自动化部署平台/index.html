前端通过Jenkins搭建自动化部署平台

发表于 2020-06-24 分类于 工程化
本文字数： 3.2k 阅读时长 ≈ 3 分钟
摘要：前端通过Jenkins搭建自动化部署平台，只需正常git push 代码，便自动打包发布

一． 目的：

  • 自动化构建，发布前端代码至服务环境，告别手动操作带来的烦琐和时间上的浪费。提升开发效率，这里不吐槽有的公司用SVN 加 WinSCP。

二． 原理：

  • Git上配置的钩子会监测push事件和push tag事件，触发配置指定的服务器IP端口下的构建脚本执行。

三． 配置步骤：

  • （步骤中已解决本次踩的坑，网上资料都写的是很简单顺利的配置完成，实际上很多坑要踩）自动化构建，发布前端代码至服务环境，告别手动操作带来的烦琐和时间上的浪费。提升开发效率。

  • 卸载本机低版本的git，自行安装更高版本的git

四. 回避坑

  1. 配置Jenkins这个过程遇到的坑最多：
  • 在系统管理=>插件管理中找到 NodeJS Plugin，勾选后点击安装。
  • 在系统管理=>Global Tool Configuration 中找到Git全局配置，这里要找到git安装目录下的/bin/git。像刚才我们使用whereis git找到目录，实际填写是/usr/local/git/bin/git。
  • 在源码管理中勾选GIT，填入项目地址，添加凭证，设置要监听的分支。
  • 在构建触发器中选择 Build when a change is pushed to GitLab，会自动根据上一步配置生成钩子链接。然后点击高级，生成secret token。

  • 为了能够与 GitLab 配合，需要进入对 GitLab进行一些设置，进入GitLab项目下，并点击settings。在设置界面单击左侧的Integrations & services，并选择add hooks。从下拉菜单中，选中Jenkins(GitLab plugin)，把上一步的钩子连接和secret token填入，点击Add Hooks保存。
  • 在构建环境中选择Provide Node & npm bin/ folder to PATH，选中刚才安装的node插件名字。
